# Transcriber GUI UX Design Specification

## 1. 设计原则

### 1.1 工作台优先

界面应服务于重复使用，而不是一次性运行。用户打开后应该立即知道要粘贴链接、在哪里看进度、在哪里拿结果。

### 1.2 平台差异显性化

B 站和抖音虽然共用转写流程，但输入链接、cookies、常见失败不同。两个 tab 应保留一致结构，同时只展示平台相关配置。

### 1.3 状态比日志重要

默认界面展示人类可读状态和下一步动作。原始日志作为可展开详情，不占据主体验。

### 1.4 结果即操作

用户完成转写后最关心“打开文案、复制文案、定位文件”。完成状态必须直接提供操作按钮。

## 2. 信息架构

主窗口分为三栏：

```text
┌────────────────────────────────────────────────────────────┐
│ Transcriber                       设置  输出目录  历史记录   │
├───────────────┬──────────────────────┬─────────────────────┤
│ 平台输入区     │ 任务队列              │ 文案/结果区           │
│               │                      │                     │
│ [抖音] [B站]   │ 任务卡 1              │ 清洗版文案             │
│ 链接输入框      │ 下载中 42%            │ 原始 ASR              │
│ Cookies 设置   │ 任务卡 2              │ 文件列表              │
│ 输出目录选择    │ 完成                  │ [复制] [打开文件夹]    │
└───────────────┴──────────────────────┴─────────────────────┘
```

第一版可使用左右两栏加底部结果区实现，但布局应保留右侧工作区扩展位。

## 3. 主窗口

### 3.1 顶部栏

内容：

- 应用名：`Transcriber`
- 当前输出目录摘要
- `设置`
- `打开输出目录`

行为：

- 点击输出目录打开文件管理器。
- 设置按钮打开设置对话框。

### 3.2 平台输入区

#### 抖音 Tab

组件：

- 多行输入框。
- placeholder：`粘贴抖音短链或分享文本，一行一个`。
- 按钮：`从剪贴板粘贴`、`解析链接`、`清空`。
- 主按钮：`开始处理`。

#### Bilibili Tab

组件：

- 多行输入框。
- placeholder：`粘贴 B 站 BV 链接、b23 短链或分享文本`。
- cookies 选择：
  - `不使用`
  - `Chrome`
  - `Edge`
- 辅助提示：`遇到 412 时选择浏览器 cookies`。
- 按钮：`从剪贴板粘贴`、`解析链接`、`清空`。
- 主按钮：`开始处理`。

### 3.3 输出目录选择

组件：

- 路径显示框。
- `选择目录`。
- `打开目录`。

规则：

- 路径过长时中间省略。
- 鼠标悬停显示完整路径。
- 路径不可写时显示错误状态。

## 4. 任务队列

任务队列以卡片或表格展示。V1 推荐卡片，因为状态和按钮更清楚。

### 4.1 任务卡字段

- 平台图标或文本：`抖音` / `B站`
- 视频标题，未解析前显示 URL 摘要。
- 当前状态。
- 进度条。
- 已用时。
- 操作按钮。

### 4.2 状态文案

| 内部状态 | 用户文案 |
| --- | --- |
| queued | 等待中 |
| parsing | 解析链接中 |
| downloading | 下载视频中 |
| merging | 合并音视频中 |
| extracting_audio | 提取音频中 |
| transcribing | 转写中 |
| writing_files | 生成文件中 |
| done | 完成 |
| failed | 失败 |
| cancelled | 已取消 |

### 4.3 操作按钮

等待中：

- 取消

下载中：

- 取消
- 查看日志

转写中：

- 查看日志
- 请求取消

完成：

- 打开校对稿
- 打开文件夹
- 复制文案

失败：

- 重试
- 查看详情
- 复制错误

## 5. 文案/结果区

当未选中任务时显示空状态：

```text
选择左侧任务后查看转写结果
```

当任务完成时显示：

- 标题
- 视频时长
- 来源链接
- 清洗版文案
- 原始 ASR
- 文件列表

Tabs：

- `清洗版`
- `原始 ASR`
- `日志`
- `文件`

### 5.1 文件列表

每行：

- 文件类型
- 文件名
- `打开`
- `定位`
- `复制路径`

文件类型：

- Markdown 校对稿
- Raw 文本
- 视频
- WAV
- Info JSON

### 5.2 文案操作

按钮：

- `复制全文`
- `打开 Markdown`
- `定位 Markdown`

V2 预留：

- `生成摘要`
- `提取标题`
- `改写成短视频脚本`

## 6. 设置对话框

设置项：

- 输出目录。
- ffmpeg 路径。
- Whisper 模型：tiny / base / small / medium。
- 默认是否保留 wav。
- B 站默认 cookies 来源。
- 完成后是否自动打开输出目录。
- 日志保留数量。

按钮：

- `保存`
- `恢复默认`
- `重新检测依赖`

## 7. 错误体验

### 7.1 B 站 412

主提示：

```text
B 站需要浏览器 Cookies
当前链接可以识别，但 B 站拒绝返回播放地址。
```

操作：

- `使用 Chrome Cookies 重试`
- `使用 Edge Cookies 重试`
- `查看错误详情`

### 7.2 ffmpeg 缺失

主提示：

```text
未找到 ffmpeg
请选择 ffmpeg.exe 或将其加入 PATH。
```

操作：

- `选择 ffmpeg.exe`
- `重新检测`

### 7.3 Whisper 模型缺失

主提示：

```text
Whisper small 模型未缓存
首次运行会自动下载模型。
```

操作：

- `继续`
- `切换模型`
- `打开缓存目录`

## 8. 视觉风格

风格应偏生产力工具：

- 背景：浅色或系统默认。
- 强调色：蓝色用于主操作，绿色用于完成，红色用于失败，黄色用于需注意。
- 卡片圆角不超过 8px。
- 字体使用系统默认，保证 Windows 下清晰。
- 不使用装饰性渐变、营销式 hero 或大面积插画。

## 9. 可访问性

- 主要按钮支持键盘焦点。
- 失败状态不能只靠颜色区分，必须有文本。
- 长路径提供 tooltip。
- 日志区域支持复制。

## 10. V1 验收

- 用户不用命令行可以完成 B 站转写。
- 用户不用命令行可以完成抖音转写。
- B 站 412 可通过 Chrome cookies 重试。
- 完成后可直接打开 Markdown 和所在文件夹。
- 任务运行时窗口不冻结。
